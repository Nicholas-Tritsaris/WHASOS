How to Build

Complete step-by-step guide to building WHAOS from source.

Prerequisites

HostUse case
Ubuntu 24.04 LTSISO build, Linux native build
Windows + WSL2VS2022 build for Linux target

Required: gcc 12+, CMake 3.20+, sudo, ~10 GB disk

1. Build C components

Option A: Visual Studio 2022

Install VS2022 with Linux development with C++, WSL2. In WSL: sudo apt install build-essential cmake ninja-build libgtk-3-dev pkg-config

Open WHAOS folder, select Linux target, Build → Build All.

Option B: Linux native (make)

make init
make desktop

2. Build the ISO (Linux only)

sudo apt-get install -y debootstrap xorriso grub-pc-bin grub-efi-amd64-bin squashfs-tools rsync wget
./scripts/build_iso.sh 1.0

Output: output/whacos-1.0-amd64.iso

3. Test in VirtualBox

Create VM (Linux 64-bit, 2 GB RAM, 20 GB disk), attach ISO, boot.

WHAOS Architecture

WHAOS is a 64-bit OS for x86-64 (AMD64), designed for VirtualBox with a Windows 7–like desktop.

1. Boot and Kernel

Boot flow: BIOS/UEFI → GRUB2 → WHAOS Kernel → init/whad

Kernel: Linux-based, hybrid monolithic. Drivers: Storage (IDE/AHCI), VMSVGA, PS/2, Intel e1000.

2. File System

Root: ext4. Layout: /bin, /boot, /etc, /home, /usr, etc.

3. GUI Stack

X11 (or Wayland) → Compositor → WHAOS Desktop (Start Menu, Taskbar, System Tray)

4. Core Apps

File Explorer, Control Center, Text Editor, Terminal, Task Monitor

WHAOS Assumptions

Build host: Ubuntu 24.04 LTS (or Debian 12, Fedora 40)

Approach: Path A – Linux kernel + minimal userspace + X11 + WHAOS desktop

VirtualBox: 7.x or 6.1+

Legal: GPL/MIT/BSD, no Windows code or assets

Building with Visual Studio 2022

Method 1: CMake (recommended)

File → Open → Folder. Select Linux/WSL target. Build → Build All.

Method 2: Solution

File → Open → WHAOS.sln. Platform: Linux. Build Solution (F7).

whad – no extra deps. whaos-panel – needs libgtk-3-dev on Linux.

Formatting (GPT + EFI)

parted -s /dev/sda mklabel gpt
parted -s /dev/sda mkpart primary fat32 1MiB 513MiB
parted -s /dev/sda mkpart primary ext4 513MiB 100%
mkfs.vfat -F 32 -n EFI /dev/sda1
mkfs.ext4 -L WHAOS_ROOT /dev/sda2

Root Layout

/bin, /boot, /dev, /etc, /home, /lib, /proc, /sbin, /usr, /var, etc.

Users and Security

init: whad (PID 1)

Users: root (UID 0), whauser (UID 1000)

Groups: root, wheel, users, audio, video

Permissions: Unix rwxrwxrwx, optional ACLs

Keyboard and Mouse Support

Kernel Level

PS/2 keyboard (atkbd), PS/2 mouse (psmouse), evdev exposes to X11

Desktop Level

Keyboard: Super L/R – Toggle Start menu

Mouse: Left click Start – open menu. Right click clock – context menu.

CLI (VBoxManage)

VBoxManage createvm --name "WHAOS" --ostype "Linux_64" --register
VBoxManage modifyvm "WHAOS" --memory 2048 --cpus 2 --vram 128
VBoxManage storagectl "WHAOS" --name "IDE" --add ide
VBoxManage storageattach "WHAOS" --storagectl "IDE" --port 1 --device 0 --type dvddrive --medium "whacos-1.0-amd64.iso"
VBoxManage modifyvm "WHAOS" --boot1 dvd --boot2 disk
